## DOMAIN KNOWLEDGE — THREAT INTELLIGENCE REFERENCE

### IOCs (Indicators of Compromise)

**Network IOCs**
- IPv4 addresses: Valid public IPs (not RFC1918: 10.x, 192.168.x, 172.16-31.x are internal).
  Private IPs in threat reports = lab/pivot example, NOT attacker infrastructure.
- Domains: Malicious domains often show DGA patterns (high entropy, random strings, unusual TLDs).
  Valid TLD required. Example DGA: tr4ns.cc, a9d3f2.xyz, update-cdn[.]net
- URLs: C2 endpoints often follow patterns: /gate.php, /upload, /check_in, /beacon
- User-Agent: Malicious beacon UAs mimic browsers but have subtle anomalies (wrong version, missing fields)

**File IOCs**
- MD5: 32 hex characters. SHA1: 40 hex characters. SHA256: 64 hex characters. Validate lengths.
- Malicious filenames often impersonate: svchost.exe, explorer.exe, lsass.exe, update.exe
- Malicious paths: %APPDATA%\Roaming\, C:\Windows\Temp\, C:\Users\Public\
- Named pipes: \\.\pipe\<name> — Cobalt Strike default: \\.\pipe\mojo.*, \\.\pipe\postex_*
- Mutexes: Unique strings created by malware to prevent re-infection

**Behavioral IOCs**
- Registry persistence: HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run\
- Scheduled tasks: Names that blend in: "Microsoft\Windows\WindowsUpdate\AUCheck"
- Services: Malicious services copy legitimate display names

### MITRE ATT&CK Framework Validation

**Technique ID Format**: T[1-9][0-9]{3} or T[1-9][0-9]{3}.[0-9]{3}
Examples of VALID IDs: T1566, T1566.001, T1059.003, T1003.001
Examples of INVALID: T1234567, T999, TXXX, T566

**14 Tactics and Key Techniques (ATT&CK v15)**

TA0001 Initial Access:
  T1566 Phishing (.001 Attachment, .002 Link, .003 Service), T1190 Exploit Public-Facing App,
  T1133 External Remote Services, T1078 Valid Accounts, T1195 Supply Chain Compromise

TA0002 Execution:
  T1059 Command/Scripting Interpreter (.001 PowerShell, .003 Windows Cmd, .004 Unix Shell, .006 Python),
  T1203 Exploitation for Client Execution, T1047 Windows Mgmt Instrumentation, T1053 Scheduled Task

TA0003 Persistence:
  T1547.001 Registry Run Keys, T1053.005 Scheduled Task, T1543.003 Windows Service,
  T1098 Account Manipulation, T1136 Create Account

TA0004 Privilege Escalation:
  T1055 Process Injection (.001 DLL Injection, .002 PE Injection, .012 Process Hollowing),
  T1068 Exploitation for Privilege Escalation, T1548 Abuse Elevation Control Mechanism,
  T1134 Access Token Manipulation

TA0005 Defense Evasion:
  T1036 Masquerading (.005 Match Legitimate Name), T1055 Process Injection,
  T1562 Impair Defenses (.001 Disable/Modify Security Tools),
  T1027 Obfuscated Files (.010 Command Obfuscation), T1070 Indicator Removal

TA0006 Credential Access:
  T1003 OS Credential Dumping (.001 LSASS Memory, .002 Security Account Manager),
  T1110 Brute Force, T1056 Input Capture (.001 Keylogging),
  T1558 Steal/Forge Kerberos Tickets (.003 Kerberoasting)

TA0007 Discovery:
  T1082 System Information Discovery, T1016 System Network Config Discovery,
  T1083 File and Directory Discovery, T1087 Account Discovery, T1057 Process Discovery

TA0008 Lateral Movement:
  T1021 Remote Services (.001 RDP, .002 SMB/Admin Shares, .004 SSH),
  T1550 Use Alternate Auth Material (.002 Pass the Hash), T1534 Internal Spearphishing

TA0009 Collection:
  T1005 Data from Local System, T1039 Data from Network Shared Drive,
  T1074 Data Staged (.001 Local, .002 Remote), T1560 Archive Collected Data

TA0011 Command and Control:
  T1071 Application Layer Protocol (.001 Web Protocols, .004 DNS),
  T1105 Ingress Tool Transfer, T1095 Non-Application Layer Protocol,
  T1573 Encrypted Channel (.002 Asymmetric Cryptography)

TA0010 Exfiltration:
  T1041 Exfiltration Over C2 Channel, T1048 Exfiltration Over Alternative Protocol,
  T1567 Exfiltration Over Web Service (.002 Exfil to Cloud Storage)

TA0040 Impact:
  T1486 Data Encrypted for Impact (ransomware), T1485 Data Destruction,
  T1490 Inhibit System Recovery, T1491 Defacement

**Common ATT&CK Mapping Errors to Catch**
- T1059.001 (PowerShell) belongs to EXECUTION, NOT Defense Evasion
- T1055 appears in BOTH Privilege Escalation AND Defense Evasion — context matters
- T1003 (Credential Dumping) ≠ T1110 (Brute Force)
- Initial Access techniques cannot appear in the Lateral Movement stage
- T1071 (C2 protocol) belongs to C&C, not Execution
- T1105 (Tool Transfer) belongs to C&C, not Execution

### Kill Chain Validation Rules

A logically valid attack sequence must satisfy:
1. At least ONE Initial Access or Execution technique at stage 1 or 2
2. Persistence is usually established BEFORE Lateral Movement
3. Privilege Escalation comes BEFORE or WITH Credential Access in most attacks
4. Lateral Movement REQUIRES prior Credential Access (pass-the-hash is the exception)
5. Collection MUST precede Exfiltration — exfil before collection = invalid
6. C2 is established in mid-to-late stages, not at the very beginning
7. Impact (ransomware, destruction) is always the LAST stage

**Red Flags**
- Exfiltration before Collection
- Lateral Movement as the first stage
- Impact in stages 1-3 without prior access
- Missing Execution entirely (rare but possible in fileless attacks)
- 8+ stages with duplicate tactics and no technique progression

### C2 Framework Fingerprints

Cobalt Strike (CS):
- Beacon: Named pipes \\.\pipe\mojo.*, \\.\pipe\postex_*, \\.\pipe\msagent_*
- Default ports: 443 (HTTPS), 80 (HTTP)
- Malleable C2: jitter, sleep, URI patterns configurable
- User-agent often: Mozilla/5.0 with slight variations

Metasploit / Meterpreter:
- Default port 4444 (staged), 443 (stageless)
- Stager binary ~few hundred bytes

Sliver C2:
- mTLS/gRPC, implant names are random words (e.g., "elegant-mushroom")
- Ports configurable, default 8443

Brute Ratel C4 (BRc4):
- Badger implant, similar to CS Beacon
- Used by APT groups since 2022

Falcon Prevent (NG-AV):
- ML on-sensor prevention before execution

- Covers: T1059 (scripting), T1566 (phishing), T1486 (ransomware execution)

- AD-integrated: detects Kerberos/NTLM anomalies in real time

### IOC Quality Checklist
When reviewing analysis, flag these issues:
1. RFC1918 IPs listed as "threat actor IPs" (10.x, 192.168.x, 172.16-31.x)
2. Hash with wrong length for stated algorithm
3. Technique ID format violation (too long, non-numeric, wrong prefix)
4. Technique assigned to wrong tactic (e.g., T1059 in Defense Evasion)
5. Kill chain violations (exfil before collection, lateral movement as first stage)
6. Unrealistic attribution confidence (claiming APT28 with only one IOC and no supporting evidence)
7. Windows-only technique (e.g., T1003.001 LSASS) assigned to Linux platform
8. Linux-only technique (e.g., /proc memory dump) assigned to Windows
9. CrowdStrike products assigned that don't match the techniques (e.g., Identity Protection for ransomware-only campaigns)
